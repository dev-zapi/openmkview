## 详细测试步骤

### 测试 1: 打开项目并浏览文件

1. 点击"📂 打开项目"
2. 输入测试目录路径（如：/home/user/test-markdown）
3. 验证：项目加载成功，文件树显示
4. 点击文件夹 → 验证：展开/折叠
5. 点击.md 文件 → 验证：内容显示

### 测试 2: Markdown 渲染和语法高亮

1. 打开包含代码块的 Markdown 文件
2. 验证：预览模式正确渲染
3. 验证：代码块有语法高亮
4. 切换到源码模式 → 验证：显示原始文本
5. 验证：源码模式也有高亮

### 测试 3: Outline 面板

1. 打开有标题的 Markdown 文件
2. 点击"📑"按钮 → 验证：Outline 面板显示
3. 验证：标题结构正确显示
4. 点击标题 → 验证：页面滚动到对应位置
5. 拖动 Outline 面板手柄 → 验证：宽度可调

### 测试 4: 文件搜索

1. 打开包含多个文件的項目
2. 点击"🔍"按钮 → 验证：搜索框显示
3. 输入搜索词 → 验证：文件树实时过滤
4. 清除搜索 → 验证：恢复完整文件树

### 测试 5: 文件创建

1. 点击"+"按钮或右键菜单
2. 输入文件名（如：test.md）
3. 点击确定 → 验证：文件创建成功
4. 验证：新文件出现在文件树中

### 测试 6: 文件重命名

1. 右键点击文件
2. 选择"重命名"
3. 输入新文件名 → 验证：重命名成功
4. 验证：文件树更新

### 测试 7: 文件删除

1. 右键点击文件
2. 选择"删除"
3. 确认删除 → 验证：文件删除成功
4. 验证：文件树更新

### 测试 8: Git 操作

1. 打开 Git 仓库项目
2. 点击"🌿"按钮 → 验证：Git 面板显示
3. 验证：文件状态正确显示
4. 点击"Stage All" → 验证：文件暂存
5. 点击"Commit" → 输入信息提交 → 验证：提交成功
6. （可选）Push/Pull 测试

### 测试 9: 设置

1. 点击"⚙️"按钮 → 验证：设置对话框显示
2. 选择 Markdown 宽度（如：70%）
3. 点击"保存" → 验证：设置生效
4. 验证：Markdown 内容宽度改变

### 测试 10: 布局调整

1. 拖动侧边栏手柄 → 验证：宽度变化（150-500px）
2. 拖动 Outline 面板手柄 → 验证：宽度变化
3. 验证：布局响应正常
